## Google「2020五月」核心演算法更新

在5/4這天，Google推出了今年第二次的核心演算法更新，代號為「May 2020 Core Update」，這波更新影響相當的劇烈，筆者所管理的其中一個網站掉了將近25%的流量。

## 網頁UX指標三本柱 – LCP / FID / CLS

網頁的用戶體驗一直以來都相當難量化，Chrome部門希望解決這一項痛點，選定了三項核心指標，並訂出各項指標的好壞標準，希望能將UX的衡量指標系統化。這三項指標分別為:

* Largest Contentful Paint (LCP): 衡量網站的速度
* First Input Delay (FID): 衡量網站的互動性
* Cumulative Layout Shift (CLS): 衡量網頁元件的視覺穩定性

![網站UX三大指標](https://www.darrenhuang.com/wp-content/uploads/2020/05/UX三大指標.jpg)
由Chrome所倡議的標準不容小覷，很多工具都相繼呼應將開始以這三項指標為軸，包括PageSpeed Insights, WebPageTest, CrUX, Lighthouse…等都宣布會在近期內開始支援，會在下面的篇幅再做介紹。

連結: [Chromium文章](https://blog.chromium.org/2020/05/introducing-web-vitals-essential-metrics.html) / [web.dev文章](https://web.dev/vitals/) / [amp.dev文章](https://blog.amp.dev/2020/05/06/amp-web-vitals-a-better-web-together/) / [(中文)iThome報導](https://www.ithome.com.tw/news/137434)

## 計算Web Vitals的Chrome插件

安裝這個Chrome插件可以在瀏覽時自動計算網頁的LCP, FID, 和CLS分數。此插件尚未在Chrome的插件商店上架，需透過以下方法安裝: 

1. [下載ZIP檔](https://github.com/googlechrome/web-vitals-extension/archive/master.zip)
2. 解加縮後得到`web-vitals-extension-master`資料夾
3. 在Chrome的網址列輸入 `chrome://extensions`
4. 將最右上角的開發者模式打開 (Enable Developer Mode)
5. 將 `web-vitals-extension-master`資料夾拖曳Chrome後安裝

![計算UX chrome插件](https://www.darrenhuang.com/wp-content/uploads/2020/05/計算UX-chrome插件.png)
連結: [GitHub頁面](https://github.com/GoogleChrome/web-vitals-extension)

## WebPageTest支援Web Vitals

WebPageTest是在測試網頁速度時非常好用的一項工具，這項工具現在將Web Vitals作為報告裡的重要指標，同時也會在網頁載入的逐格影片中秀出Chrome所判定的LCP，並用紅色線條框起來。


> The synthetic Web Vitals (LCP, CLS and TBT) are now front\-and center in the WebPageTest results for browsers that report them (Chromium\-based).  
>   
> The Filmstrip view also shows a red box around the frame that corresponds to LCP so you can verify it measures correctly for your site. [pic.twitter.com/cevy8PDOGW](https://t.co/cevy8PDOGW)
> 
> — Patrick Meenan (@patmeenan) [May 12, 2020](https://twitter.com/patmeenan/status/1260294231589113859?ref_src=twsrc%5Etfw)

註: FID沒辦法透過測試工具計算，因為工具並不會跟網頁有”互動”，所以用Lab Data一般會用Total Blocking Time (TBT)取代。TBT是另一個指標(web vital)，但並非Chrome Team所定義的核心指標(web core vital)。

連結: [WebPageTest工具](https://webpagetest.org/)

## CrUX報告紀錄Web Vitals

Chrome User Experience (CrUX)是一份公開的database，紀錄了Chrome用戶在瀏覽網頁時的網頁表現，相比於一般的測量工具的Lab Data，CrUX的資料屬於Field Data，相當珍貴。而這項公開資料也開始記錄三項web core vitals，可以利用Google BigQuery跑出來，以下為104人力銀行網站的結果。

![104人力銀行網站用戶體驗](https://www.darrenhuang.com/wp-content/uploads/2020/05/104人力銀行網站用戶體驗-1024x665.png)
上述的資料可以解讀為：92\.08%的使用者在瀏覽104人力銀行網站(www.104\.com.tw)時的LCP載入時間是好的(\<2500ms)，4\.97%普通(\>\=2500ms, \<4000ms)，2\.91%不好(\>\=4000ms)，其中第75個百分位數的LCP為1400ms。

連結: [跑出上圖結果的Query](https://github.com/GoogleChrome/CrUX/blob/master/sql/core-web-vitals.sql)

## LinkedIn整個網站從Google搜尋消失

在5/6這天，www.linkedin.com整個網站從Google上消失，將近兩億個網頁和Knowledge Panel上的圖示都不見了。

![linkedin從Google搜尋消失](https://www.darrenhuang.com/wp-content/uploads/2020/05/linkedin從Google搜尋消失.png)
John Mueller隨後在發了一條推特說:「用GSC裡的移除工具移除”http://”的版本，會移除所有的變體(http/https/www/non\-www)。」雖然沒有指名道姓， 但Linkedin這個悲劇應該就是由GSC的移除工具所造成的了。


> PSA: Removing the "http://" version of your site will remove all variations (http/https/www/non\-www). Don't use the removal tools for canonicalization.<https://t.co/yTfRzWZGtd>
> 
> — John 🧀 … 🧀 (@JohnMu) [May 6, 2020](https://twitter.com/JohnMu/status/1258011921636302848?ref_src=twsrc%5Etfw)

## Google的Question Hub計畫

Google有一個Question Hub計畫，蒐集使用者在Google上沒找到滿意答案的問題，整理在一個地方，讓線上媒體看到後以此創造合適的內容。這是相當實用的一項工具，內容生產者可以看到網路上尚未被回答的問題，而且還是來自真正的人!

這項計畫去年在印度推出，現在還在Beta測試中，而最近在美國也有準備推出的跡象，筆者人在美國，在Google搜尋時常常會在SERP下方被問到我的問題是否有被回答到，如果沒有，我的問題是什麼。

![google question hub計畫](https://www.darrenhuang.com/wp-content/uploads/2020/05/google-question-hub計畫.png)
這是相當值得關注的一項計畫，Search Engine Rountable有印度用戶提供的更多節圖。

連結: [Google Question Hub](https://questionhub.withgoogle.com/intl/en/) / SER報導([1](https://www.seroundtable.com/google-question-hub-28020.html), [2](https://www.seroundtable.com/google-search-console-content-creators-invite-29339.html), [3](https://www.seroundtable.com/google-question-hub-screen-shots-29395.html), [4](https://www.seroundtable.com/google-questions-hub-us-for-covid-19-queries-29334.html))
