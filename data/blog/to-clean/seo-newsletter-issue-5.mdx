## 「Top Questions Answered 」\- SERP上的名人影音Q\&A

## Safari全面阻擋第三方Cookies

從Safari 13\.1開始，Safari將”全面”封鎖第三方Cookies。在官方的公告中有特別提及Tom Anthony以前的一篇文章，講述網站可以透過簡單的Javascript來檢查用戶是否有登入Facebook、Google、Twitter…等等，這項伎倆在這次的更新後也將失效。 

Tom Anthony是筆者很仰慕的一位SEO，聽過他多次演講，也曾在數位引擎上[介紹過](https://www.facebook.com/searchenginecommunity/posts/257231775014772)他的另一項事蹟!

連結: [WebKit公告](https://webkit.org/blog/10218/full-third-party-cookie-blocking-and-more/) / [Tom Anthony文章](https://www.tomanthony.co.uk/blog/detect-visitor-social-networks/) / [SEL報導](https://www.searchenginejournal.com/safari-full-3rd-party-cookie-blocking/357198/)

## Bing推出網址/內容提交API

Bing推出這兩個API，讓站長們可以在新網頁推出或網頁內容更動時，即時將這些改動主動提交給Bing知道，不需要等待Bingbot自己來抓取跟索引。之前兩個API都在封測，但現在URL Submission API已經開放所有人使用。

網址提交API的上限是每天一萬個URLs，且可以提交3XX或4XX的網址讓Bing知道網站上的更動。

```
JSON request sample: 
POST /webmaster/api.svc/json/SubmitUrlbatch?​apikey=sampleapikeyEDECC1EA4AE341CC8B6 HTTP/1.1​
Content-Type: application/json; charset=utf-8​
Host: ssl.bing.com​
{
"siteUrl":"http://yoursite.com",​
"urlList":[
"http://yoursite.com/url1",
"http://yoursite.com/url2",
"http://yoursite.com/url3"
]
}
```
連結: [Bing URL/Content Submission API網站](https://www.bing.com/webmasters/url-submission-api)

## Schema.org 7\.0發布

Schema 7\.0版本推出了幾項和武漢肺炎有關的標記與更新，例如給學校或其他重要機關使用的”SpecialAnnouncement”；標記測試場所的”CovidTestingFacility”；註明活動場所為線上或線下的”eventAttendanceMode”。

## Google我的商家在疫情期間的政策調整

許多地方商家因為這次疫情的影響必須暫時歇業，而Google也做出了以下幾項政策調整，來因應這樣的現象。

1\.) **允許商家將自己標示為「暫時歇業」**– 以往只有Google客服可以將商家加上這個標記，現在允許店家自己標示。

2\.) **透過對話機器人(Duplex)來確認並更新商家的營業時間** – 這個相當酷，如果還不知道Duplex可以聽聽看下面這段語音。


Duplex預約剪頭髮服務

3\.) **暫停開放評論與問與答、商家資訊編輯、及新商家驗證** – 將人力集中投資在衛生相關商家的資訊，而暫停其他優先度較低的服務項目。

## Chrome 81將開始檔掉HTTP載入的圖片資源

Chrome 81 (預計2020\.04\.07發布)開始，對於用HTTP載入的圖片，Chrome將會自動升級成HTTPS，如果HTTPS抓取資源失敗，那圖片就**不會**被載入。

Mixed Content是SEO裡很常見的問題，尤其是圖片透過CDN來提供，而非放在相同網域下時，站長只記得要幫自己網域裝上HTTPS而忘記了CDN伺服器。建議大家花點時間檢查網站有沒有這樣的問題。 


> In the stable version of Chrome now (v80\), audio and video are currently being blocked if they can't be auto\-upgraded to https. With Chrome 81 (which hasn't rolled out yet), images will be blocked from loading if they can't auto\-upgrade to https <https://t.co/lE2LfR9Xv5> [pic.twitter.com/Qyyh4GUjNH](https://t.co/Qyyh4GUjNH)
> 
> — Glenn Gabe (@glenngabe) [March 23, 2020](https://twitter.com/glenngabe/status/1242072232198840323?ref_src=twsrc%5Etfw)

Bonus: 感謝詳細閱讀這份電子報的你! 如果不知道如何檢查自家網站有沒有Mixed Content的問題，歡迎到數位引擎留言，讓筆者幫你免費看看\~

連結: [Chromium公告](https://blog.chromium.org/2019/10/no-more-mixed-messages-about-https.html#:~:text=In Chrome 81, mixed images will be autoupgraded to https://, and Chrome will block them by default if they fail to load over https://. Chrome 81 will be released to early release channels in February 2020.) / [相關文章](https://www.gsqi.com/marketing-blog/google-chrome-block-mixed-content/)
