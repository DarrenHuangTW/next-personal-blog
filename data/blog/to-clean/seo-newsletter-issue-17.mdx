## Google支援shippingDetails標記

Google現在開始支援*shippingDetails*這項Schema屬性，讓賣家能夠將商品的送貨資訊呈現在搜尋結果上。

![送貨細節 結構化資料](https://www.darrenhuang.com/wp-content/uploads/2020/09/google產品運送細節結構化資料.png)
沒有Google Merchant Center的店家也能透過Schema Markup讓運費資訊出現在搜尋結果。

以下為簡單的範例，官方的指南中有很多更細節的標記，包括「*每日收單時間*」、「*那些地區不在寄送範圍*」、「*訂單處理工作日*」…等等。

```
# 範例 - 標記的商品寄送到紐約的運費為0
"shippingDetails": {
  "@type": "OfferShippingDetails",
  "shippingRate": {
    "@type": "MonetaryAmount",
    "value": "0",
    "currency": "USD"
  },
  "shippingDestination": [{
    "@type": "DefinedRegion",
    "addressCountry": "US",
    "addressRegion": ["NY"]
  }]
}
```
連結: [官方公告](https://webmasters.googleblog.com/2020/09/new-schemaorg-support-for-retailer.html) / [產品頁結構化資料](https://developers.google.com/search/docs/data-types/product)

## Googlebot將開始以HTTP/2進行抓取

**從2020年11月起，Googlebot將以HTTP/2 (h2\)抓取部分網站。**

以HTTP/2抓取能夠更有效率，節省Google的資源。即使伺服器支援，站長也沒有辦法強制讓Googlebot以h2抓取，是否從HTTP/1\.1升級到HTTP/2由Google自行決定。

當Google決定以h2抓取你的網站後，站長會在GSC中收到通知。此外，也可以透過server logs來看到Googlebot以何種方式抓取。

**網站支援h2最主要的優勢是速度變快以及節省資源，並不會影響到網站的索引與排名。**

## 官方文章: 搜尋結果如何即時反映時事

Danny Sullivan最近花了很多篇幅解釋Google背後的運作方式，但這一篇我覺得滿有趣的，而且剛好是最近親身經歷的事情所以簡單翻譯一下。

![google依據時間地點給出不同結果](https://www.darrenhuang.com/wp-content/uploads/2020/09/Google反映時事.png)
「*天空為何是橘色的?*」搜尋結果

「*天空為何是橘色的?*」，這個搜尋字在九月中突然有超多人搜尋，因為灣區的天空因為野火，發生了這個奇怪的現象。

語言理解是搜尋的核心，所以Google演算法在遇到這種前所未見的情形時，就仰賴其他資訊(例如搜尋者的時間和地點)，來補上搜尋意圖背後的重要前後文(critical context)，

在這個例子中，**Google的新鮮度指標(freshness indicators)判斷這個字屬於突發事件，需要將「較新」且「在地」的資訊提供給使用者，而非以往排名在這個字的頁面，因為即便搜尋字相同，搜尋者的搜尋意圖也和原本不同了**，過去可能是要了解黃昏的原因，但現在卻是要了解這個特殊現象。

也許與一般認知的SEO沒有太大關係，但[文章](https://blog.google/products/search/why-sky-orange-how-google-gave-people-right-info/)滿有趣的，推薦閱讀
