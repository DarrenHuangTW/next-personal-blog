## 官方公告: Core Web Vitals 更新延遲至六月中

Google 發了一份官方公告，說明很多人引頸期盼的 Core Web Vitals 更新，將從原定的五月份改為**六月中逐步推出，預計在今年八月底全部完成**。關於CWVs的自從公告以來已經介紹過非常多次，如果還對此懵懵懂懂可以參考[之前寫的文章](https://www.darrenhuang.com/core-web-vitals-lcp-fid-cls.html)，以及上其提到的[官方Q\&A](http://43.201.249.211/seo-newsletter-issue-29.html#toc-9)。

## GSC 將開放讓用戶回報索引問題 (US Only)

Google 將開放讓使用者能夠回報網頁索引的問題，可以透過GSC，或是透過[這個表格](https://support.google.com/webmasters/contact/indexing_issue_form)。公告中說只開放給美國用戶，不確定其他國家是否能打開表格或者適用。


> We are currently piloting this in the US only and it should be fully available to all in the US within a week or less. We will reassess the usefulness of a support expansion in this direction in a few weeks.
> 
> — Google Search Central (@googlesearchc) [April 28, 2021](https://twitter.com/googlesearchc/status/1387460807831683072?ref_src=twsrc%5Etfw)

 「 網頁沒被收錄(not indexed)」與「網頁沒有排名(not ranking)」是兩碼子事，我遇過號稱”沒收錄”情形，幾乎都其實是沒有排名；或者網頁本身出了錯，而GSC中的[網頁檢測工具](https://support.google.com/webmasters/answer/9012289#missing_page)能判斷出大部分的錯誤，我個人認為這次推出的回報機制真正派上用場的時機不多。

## Google 2020 Webspam報告

Google 最近發布了年度 webspam report，告訴大家他們在打擊垃圾內容上的一些進度，一個很不可置信的數字是其每天發現400億個垃圾網頁! 報告當中講了他們的簡單流程 \-首先會在爬取階段Google就有一套系統，避免垃圾內容被收錄；如果不小心收錄了垃圾內容，排名機制也會防止他們排在搜尋結果的前面，並用這個資訊來優化抓取階段的機器學習模型；最後才是人工的審查。

除了垃圾內容以外，自2020年開始Google也擴大打擊其他種類的有害內容，包含詐欺、誹謗…等等，甚至也包括最近的[「商品評論」更新](http://43.201.249.211/seo-newsletter-issue-30.html#toc-2)。

![WebspamReport2020 EverySteps](http://43.201.249.211/wp-content/uploads/2021/04/WebspamReport2020_EverySteps-1024x657.png)
垃圾內容防制流程

Google 每年都會出這樣的一份報告，數位引擎以前也有介紹過去幾年的報告，歡迎閱讀!

## 安布思沛統計: 各行各業的CWV分數

電通集團旗下的安布思沛(iProspect)做了這項統計，取英國15個行業領域的各前100名流量的網站，共1,500個網站，並分析他們的[Core Web Vitals分數](https://www.darrenhuang.com/core-web-vitals-lcp-fid-cls.html)。

## [官方文章] Google 什麼情況為何會從搜尋結果上移除網頁

Danny Sullivan 在[這篇文章](https://blog.google/products/search/when-and-why-we-remove-content-google-search-results/)中講解大部分的人比較少、也不想遇到的問題：「怎麼讓某個網頁在Google上被移除? 」，一般分為三種情況。

1\. **法律因素 (ex :版權、合法性)**Google 並不是執法機構，法律也根據地區有所不同，因此像是版權、原創內容…這類型的爭議，需要透過執法機關裁定並通知Google把內容移除，而不是抱怨了Google就會調查，除非是項兒童色情或毒品這種明顯違法的情況。

![侵權下架Google搜尋](http://43.201.249.211/wp-content/uploads/2021/01/侵權下架Google搜尋-1024x498.png)
*[範例](https://lumendatabase.org/notices/18982842): 網站「瘋先生」控告另一個網頁抄襲，使該網頁在Google Search上被移除。*

**2\. 個人申訴**雖然對於需要法律裁定的東西Google並不主動受理，但像個人機密資料外洩，例如鄉民大量起底把某人的身分證字號、聯絡資料、就醫紀錄全部翻出來放上網的情況，還是可以向[Google申述](https://support.google.com/websearch/troubleshooter/9685456)請求移除。但如果是商家負面評論或是有誹謗嫌疑的內容，就可能不是受理的範圍。

**3\. 規模化**  
要能處理海量的申訴，建立一個可以規模化、自動化的機制還是必要的，Google會利用申述來優化自己的品質監管系統，舉例來說，如果一個網站有大量的違反著作權通知，那Google就會以此為質量訊號在搜尋結果調整該網站的埋名和曝光。

連結: [官方文章](https://blog.google/products/search/when-and-why-we-remove-content-google-search-results/)

## [影片] SEO現況分析 from 網路年鑑報告

[Web Almanac](https://almanac.httparchive.org/en/2020/table-of-contents) (網路年鑑) 是 HTTP Archive 這個組織每年會出的一份報告，今年為第二年，這支影片由 Aleyda Solis 來講述裡面SEO章節的一些發現，像是…

* 99%的網頁有使用title tag，中位數是六個字，38個字元 (去年只有20個字元!)
* 只有68%左右的網頁有meta description
* 28\~30%左右的網頁有使用`rel=nofollow`在連結上，但`rel=ugc`和`rel=sponsored`的使用率只在0\.3%上下
* HTTPS的使用率已經達到73\~77%，比去年高出10%
* 不到1%的網頁有使用AMP


上面僅為一小部分，這份網路年鑑報告除了SEO以外，含有 CSS, JavaScript, SEO, Performance, CMS, Ecommerce…一共22個章節，其他的章節也有一些有趣的資訊，例如在 [CMS](https://almanac.httparchive.org/en/2020/cms) 和 [Ecommerce](https://almanac.httparchive.org/en/2020/ecommerce) 的章節，可以分別看到現在最常見內容或電商平台有那些、每一個的平均CWV分數為和、AMP的使用率…等等，推薦大家有空過去翻翻。
