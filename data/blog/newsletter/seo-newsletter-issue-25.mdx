## 搜尋結果呈現商品「降價」資訊

這個搜尋結果目前僅在美國的英文搜尋結果出現，**降價資訊是 Google 根據商品結構化資料的過去價格紀錄 ，自動生成的**。另外，要有這樣的結果，Schema Markup 中 `Offer` 標籤的 `price` 屬性必須是固定的金額而非一個價格範圍。

[過去電子報](https://www.darrenhuang.com/seo-newsletter-issue-17.html#toc-6)曾介紹 Google 開始支援更多與商品有關的 Schema 標記 (ex: 運送細節)，現在甚至追蹤過去的價格，讓一般搜尋結果也能像 Google Shopping 一樣幫助使用者[比較價格](https://www.darrenhuang.com/seo-newsletter-issue-19.html#toc-5)。

![Google 降價 搜尋結果](http://43.201.249.211/wp-content/uploads/2021/01/Google-降價-搜尋結果.png)

## Chrome v88 幾項更新

這次的 Chrome 88更新，有幾個相當不錯的重點；

1\.) **擋掉HTTPS頁面上的HTTP資源** — 尤其是圖片透過CDN或放在子網域的網站要注意這點，很常看到只有主網站裝上HTTPS，但忘記其他子資源所在位置的安全憑證，所造成的 Mixed Content 問題。

![mixed content chrome 警告](http://43.201.249.211/wp-content/uploads/2021/01/mixed-content-chrome-警告-1024x348.png)
圖片來源: [Google Chrome 改版資訊](https://support.google.com/chrome/a/answer/7679408)

**2\.) 更不惱人的許可請求** — 現在太多網站濫用通知功能了，動不動就要知道你的位置資訊或想發送推播通知，這個更新是我最喜歡的一點! 要開啟這項功能可到這個網址將其打開: [chrome://flags/\#permission\-chip](chrome://flags/#permission-chip)

![chrome 推播通知](http://43.201.249.211/wp-content/uploads/2021/01/chrome-推播通知.png)
範例: 新版的許可請求 ([圖片來源: Chrome Unboxed](https://chromeunboxed.com/whats-new-chrome-88))

**3\.) 分頁搜尋功能** — 能夠一次搜尋所有分頁上的內容，對於工作上容易不小心就越開越多分頁的人很有幫助。要開啟這項功能可到這個網址將其打開: [chrome://flags/\#enable\-tab\-search](chrome://flags/#enable-tab-search)

![Chrome 分頁搜尋](http://43.201.249.211/wp-content/uploads/2021/01/Chrome-分頁搜尋-1024x308.png)
全分頁搜尋

**4\.) 開發者工具加入網頁體驗指標** — 在開發者工具中多一個[網頁體驗指標](https://www.darrenhuang.com/core-web-vitals-lcp-fid-cls.html)的選項，搭配截圖可以清楚看到在載入的哪個環節被判定為FCP、LCP、或有版位移動的情形，相當實用。

![chrome 88 網頁體驗指標](http://43.201.249.211/wp-content/uploads/2021/01/chrome-88-網頁體驗指標-1024x596.png)
Chrome DevTools \> Performance \> Web Vitals

## GA4 – Cohort Analysis 介紹 \& Demo 帳戶開放

GA4中的同類群組分析(Cohort Analysis)，除了原本的*標準 (standard)*，現在新增了兩種計算方式 – *累計 (rolling)*、*累積 (cumulative)*。同類群組分析是相當重要的分析方法，可以幫助分析者看出**有完成特定事件/行為的使用者，是否對於網站的某項指標帶來顯著的影響**，對於這個概念還不熟悉的話，非常推薦大家閱讀這篇Max行銷誌上的文章。

## 官方Podcast – Search Off the Road EP.12

這集的 Podcast 請來了 Cherry Prommawin，她是負責亞洲區的 Google 搜尋質量小組成員，也是去年台灣舉辦 [Google 搜尋基礎研討會](https://publishersonair.withgoogle.com/events/search-basics-tw)時，幫忙回答問題的那位。這集的第一個重點是，**Cherry 提到 Google 在東南亞遇到的一個問題是，在一些語言、或特定領域中，它們沒有足夠的內容來回傳給使用者，畢竟這個世界上有很多相當少人在使用的語言(或文字)**，所以她工作的其中一部份就是要鼓勵大家創造內容。

Gary補充說道。**索引的階段有一個「Index Selection」的過程，他們不會因為某種語言的使用人數少，就花費較少的資源索引**，而事實上，Google內部有個機制就是要確保不會有這樣子的情況發生。

另一個有趣的重點是，**Gary提到Google的索引其實是有等級之分的**。因為網路世界非常大，Google的索引系統也需要相當大的空間來儲存，但絕大多數的網頁可能發布後就不再更新，平時也不會被搜到，因此這類型的”檔案”會被放在儲存成本低但速度慢的空間中，例如 SSD (固態硬碟)；相較之下，像新聞類型的網頁，可能每秒都會出現在搜尋結果好幾次，這類型的檔案就會被放在儲存成本高但速度快的空間中，例如 RAM (隨機存取記憶體)。


連結: [逐字稿](http://traffic.libsyn.com/preview/search-off-the-record/Search_Off_the_Record_-_12th_episode.pdf) / [SER報導 (search indexing tiers)](https://www.seroundtable.com/google-search-indexing-tiers-30784.html) / [SER 報導 (google language diversity)](https://www.seroundtable.com/google-search-index-diversity-languages-30783.html)

## 【官方新手影片】從Google上移除違法內容

這部影片介紹了怎麼從 Google 上移除「違法」內容 (違反法律、釣魚網站、未授權內容…等)。所有的程序都可以從[這個網頁](https://support.google.com/legal/answer/3110420)上開始，而「移除」並不限定搜尋，可以是 YouTube、Google Photos、Google Drive、或其他 Google 產品。


Google 另外有一個 [Transparency Report 的網站](https://transparencyreport.google.com/copyright/overview)，可以看到他們至今移除了將近50億個網頁、312萬個網域，那些網頁因為侵權行為被拿掉也可以在上面找到。
