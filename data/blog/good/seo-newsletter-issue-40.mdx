## [新聞]網頁體驗更新完全推出

## [新聞] Bing內容提交API開放使用

在[第5期](https://www.darrenhuang.com/seo-newsletter-issue-5.html#toc-3)和[第13期](https://www.darrenhuang.com/seo-newsletter-issue-13.html#toc-4)有提過Bing的這兩支API，讓站長能直接將網址或是內容提交給Bing，使其能夠及時被索引。網址提交API已經推出一陣子並且有官方[WordPress外掛](https://wordpress.org/plugins/bing-webmaster-tools/)可以整合；內容提交API則是今年九月才正式開放全面使用。

![Bing 網址內容提交API](http://43.201.249.211/wp-content/uploads/2021/09/Bing-網址內容提交API.png)
[官方網站](https://www.bing.com/webmasters/url-submission-api)

兩者的差異主要在於，網址提交API是把URL提交出去請Bing來爬；而內容提交API則是把內容也放在API中直接提交給Bing (理論上Bing甚至不用派爬蟲到該網頁上)。

之前採訪Bing發言人的一段影片[曾說過](https://www.darrenhuang.com/seo-newsletter-issue-19.html#toc-8)，Bing想要將目標放在協助站長們主動提交新的或更新的內容，因為仰賴爬蟲在網路世界中不斷發掘新的網頁以及網頁內容是否有變化(Googlebot的方法)，太沒有效率了。

連結: [官方公告](https://blogs.bing.com/webmaster/september-2021/Bing-Content-Submission-API-open-for-all-webmasters)

## [Python] 將GSC資料寫入BigQuery

將GSC的資料存入BigQuery中有幾大好處，最重要的兩個莫過於 (1\.)可以擁有超過GSC預設提供的16個月資料，以及 (2\.)往後要拉資料的彈性與效率。[這篇文章](https://importsem.com/use-python-to-create-a-gsc-to-bigquery-pipeline/)分享了如何透過Python來完成這項動作。

![GSC資料結合BigQuery](http://43.201.249.211/wp-content/uploads/2021/09/GSC資料結合BigQuery.png)
範例

這個技能在以前GSC只會提供90天資料的時候非常實用，不然就只能靠[Keylime Toolbox](https://www.keylimetoolbox.com/)這類型的付費工具。現在Google雖然提供了16個月，但相信很多人還是覺得不足。整個步驟相當不複雜，推薦大家可以嘗試試試。

美中不足的是該作者的script並沒有包括`query`這個最重要的部分，雖然我還沒實作，但多寫入一欄資料而已應該不會太難，下面放上幾個類似的文章，雖然有點舊但在debug的時候應該是不錯的參考資料。

![試算表 移除HTML標籤](http://43.201.249.211/wp-content/uploads/2021/09/試算表-移除HTML標籤.png)
`=getUrlsFromSitemap(sitemapUrl)` — 返回一個XML Sitemap中的網頁

## [自我檢測] 15項Technical SEO問題及檢測\&解決方法

[這篇文章](https://seosly.com/example-technical-seo-issues/)列了15個網站在技術上會遇到的SEO問題。過去我比較少分享這類型的文章，因為會遇到的問題絕對不止這15種，這次推薦這篇文章是因為它非常地巨細靡遺，不只列了問題，也包含怎麼檢查自家網站是否有這樣的問題，以及解決方法，適合拿來做自我檢測，或就算只是讀過去也能吸收到一些知識。

![技術SEO 檢查canonical](http://43.201.249.211/wp-content/uploads/2021/09/技術SEO-檢查canonical.png)
舉例來說，在第四點「有問題的canonical」中，作者就列了幾乎所有的情境

文中列的15的問題如下: 

```
1. CWVs在通過檢測工具(lab)，但實際資料(field)卻沒通過。
2. robots.txt 擋掉網站的資源 (css, javascript, etc)
3. XML sitemap包含不必要的網址
4. 有問題的canonical
5. HTML \<head\> 與 HTTP Header 中互相衝突的指令
6. Nofollowed 或 disallowed 的內部連結
7. 未優化的內部連結
8. 沒有連出或是連入連結的網頁
9. 內部或外部連結失效
10. 內部連結有redirect
11. 無效或錯誤的hreflang tag
12. 網頁\<head\>中有無效的元件
13. HTTP vs HTTPs
14. Mixed content或內部連結使用HTTP
15. 因網站架構產生的重複內容網頁
```
連結: [SEOSLY文章](https://seosly.com/example-technical-seo-issues/)

## [摘要] Google Search Podcast – 搜尋引擎最佳化(SEO)入門指南

這集的podcast請來[Lizzi](https://twitter.com/okaylizzi)和[Michael](https://twitter.com/wysz)，兩位Google搜尋文檔的寫手，來一起討論[SEO Starter Guide](https://developers.google.com/search/docs/beginner/seo-starter-guide)這份文件。相信絕大多數的讀者都曾經看過這份新手指南，但可能很久沒回頭複習，包括我自己也是。這集比較輕鬆，主要就是聽幾位Googler談論關於這份文件的一些軼事。


1. 這份指南的誕生是在2008年，因為Google太常被問到關於SEO的建議，因此Matt Cutts建議推出一份他們對SEO的建議。
2. 早期韓國政府的官方內部文件特別提到，因為安全考量，建議網站在robots.txt中擋掉所有robots，造成Google沒有辦法抓取政府的網頁。
3. 這份新手指南中有一段在講[社群媒體](https://developers.google.com/search/docs/beginner/seo-starter-guide?hl=sr#know-about-social-media-sites)，但這段比較屬於一般行銷的範疇，幾位在討論是否該刪掉這段。
4. Gary說現在大部分社群媒體上連結都有`nofollow`屬性，但他可以想像就算有些社群網站沒有裝，**Google也會把這些平台出去的連結在內部標記成nofollow** (這一點滿有意思的)。
5. 另外一個有趣的點是Gary說，他們會收到Google內部SEO team提供的title建議，告訴他們要怎麼寫，包含每個字的開頭用大寫，然後最後放上個 `| + 網站名稱`，他覺得這些建議對沒太大的意義，但也許是和使用者體驗有關。
6. 新手指南中用了Brandon’s Baseball Cards這個網站舉例，是因為Brandon是這份指南第一版的寫手，而且https://www.brandonsbaseballcards.com/會連到Google首頁。
